# FitCute 项目推进规划

## 一、产品定位

**FitCute —— 多Agent驱动的个性化减脂决策系统**

通过多Agent协作与数据闭环优化，构建一个能够**持续学习用户行为并动态调整减脂策略**的智能教练系统。

### 核心差异

| 维度 | 传统减肥App | FitCute |
|------|------------|---------|
| 产品形态 | 工具 | 智能决策系统 |
| 计划方式 | 固定模板 | 实时动态生成 |
| 用户角色 | 主动执行 | AI主导+用户协同 |
| 个性化 | 弱 | 强（基于数据闭环） |
| 行为管理 | 无 | 主动干预 |

---

## 二、系统架构

```
[用户交互层]
      ↓
[决策编排层（Planner Agent）]
      ↓
[多Agent执行层]
      ↓
[RAG知识层]
      ↓
[数据层（用户状态）]
```

---

## 三、Agent体系

| 类型 | Agent | 职责 |
|------|-------|------|
| 调度层 | Planner Agent | 任务拆解 & 决策编排 |
| 执行层 | Diet Agent | 饮食推荐 |
| 执行层 | Workout Agent | 运动规划 |
| 行为层 | Behavior Agent | 行为干预 |
| 分析层 | Progress Agent | 数据分析 |
| 进化层 | Reflect Agent | 策略优化 |

---

## 四、开发阶段规划

### 第一阶段：基础架构（优先级：高）

搭建系统基础架构，为后续Agent开发提供支撑。

| 序号 | 模块 | 内容 | 预估工时 |
|------|------|------|----------|
| 1.1 | 数据库设计 | 用户画像、身体指标、饮食、运动、行为数据表 | 2h |
| 1.2 | 后端基础框架 | FastAPI + SQLAlchemy + JWT认证 | 3h |
| 1.3 | 用户系统 | 注册/登录/个人信息管理 | 2h |
| 1.4 | 数据记录API | 体重、饮食、运动数据记录接口 | 3h |

**预估工时：10小时**

---

### 第二阶段：RAG知识层（优先级：高）

构建知识库系统，为Agent提供决策依据。

| 序号 | 模块 | 内容 | 预估工时 |
|------|------|------|----------|
| 2.1 | 饮食知识库 | 食物热量、营养结构、减脂餐策略 | 3h |
| 2.2 | 运动知识库 | 动作库、训练模板、热量消耗模型 | 3h |
| 2.3 | 行为心理库 | 干预策略、激励机制、情绪模型 | 2h |
| 2.4 | 向量检索 | PGVector集成、Embedding检索 | 4h |

**预估工时：12小时**

---

### 第三阶段：核心Agent开发（优先级：高）

实现MVP阶段的核心Agent。

| 序号 | Agent | 功能 | 预估工时 |
|------|-------|------|----------|
| 3.1 | Planner Agent | 任务拆解、决策编排、减脂阶段判断 | 4h |
| 3.2 | Diet Agent | 每日热量目标、三餐推荐、外卖策略 | 4h |
| 3.3 | Workout Agent | 周训练计划、动作推荐、强度调整 | 4h |
| 3.4 | AI对话接口 | 用户与Agent交互的统一入口 | 3h |

**预估工时：15小时**

---

### 第四阶段：进阶Agent开发（优先级：中）

实现行为干预和进度分析功能。

| 序号 | Agent | 功能 | 预估工时 |
|------|-------|------|----------|
| 4.1 | Behavior Agent | 行为干预、触发机制、AI劝导 | 4h |
| 4.2 | Progress Agent | 体重趋势分析、平台期检测、目标评估 | 3h |
| 4.3 | Reflect Agent | 策略有效性判断、自动调整机制 | 3h |

**预估工时：10小时**

---

### 第五阶段：数据闭环与优化（优先级：中）

| 5.2 | 平台期检测 | 趋势分析、阶段判断 | 2h |
| 5.3 | 策略演化 | 个性化策略自动优化 | 3h |

**预估工时：7小时**

## 六、技术栈
