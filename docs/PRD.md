# 📄 产品需求文档（PRD）

## 🧠 项目名称

**FitCute —— 多Agent驱动的个性化减脂决策系统**

---

# 1️⃣ 产品概述

## 🎯 产品愿景

通过多Agent协作与数据闭环优化，构建一个能够**持续学习用户行为并动态调整减脂策略**的智能教练系统。

---

## 🧩 核心价值

FitMind AI 不只是“记录工具”，而是：

> 👉 **一个具备感知 → 决策 → 干预 → 进化能力的智能系统**

---

## 💡 核心差异（升级版）

| 维度   | 传统减肥App | FitMind AI |
| ---- | ------- | ---------- |
| 产品形态 | 工具      | 智能决策系统     |
| 计划方式 | 固定模板    | 实时动态生成     |
| 用户角色 | 主动执行    | AI主导+用户协同  |
| 个性化  | 弱       | 强（基于数据闭环）  |
| 行为管理 | 无       | 主动干预       |

---

# 2️⃣ 系统架构设计（核心）

## 🧠 分层架构

```
[用户交互层]
  ↓
[决策编排层（Planner）]
  ↓
[多Agent执行层]
  ↓
[RAG知识层]
  ↓
[数据层（用户状态）]
```

---

## 🧩 Agent体系设计

### Agent分类

| 类型  | Agent          | 职责          |
| --- | -------------- | ----------- |
| 调度层 | Planner Agent  | 任务拆解 & 决策编排 |
| 执行层 | Diet Agent     | 饮食推荐        |
| 执行层 | Workout Agent  | 运动规划        |
| 行为层 | Behavior Agent | 行为干预        |
| 分析层 | Progress Agent | 数据分析        |
| 进化层 | Reflect Agent  | 策略优化        |

---

## 🔄 多Agent协作流程（强化版）

```
Step1：用户输入（目标/体重/行为）
Step2：Planner解析任务（减脂阶段判断）
Step3：任务拆解：
    - 饮食策略
    - 运动策略
    - 进度分析
Step4：并行执行（Diet / Workout / Progress）
Step5：Behavior Agent判断是否干预
Step6：Reflect Agent评估是否调整策略
Step7：输出个性化“今日方案”
```

👉 面试关键点：

* 并行Agent（体现工程能力）
* Reflect（体现“自进化系统”）

---

# 3️⃣ RAG知识增强系统

## 📚 知识库结构

### 🍽️ 饮食库

* 食物热量
* 营养结构
* 减脂餐策略

### 🏋️ 运动库

* 动作库（标准 + 风险）
* 训练模板
* 热量消耗模型

### 🧠 行为心理库

* 拖延干预策略
* 激励机制
* 情绪进食模型

---

## 🔍 检索流程

```
用户上下文 → Embedding → 向量检索 → Top-K → Agent决策
```

---

## ⚙️ 技术选型

* Embedding：OpenAI / BGE
* 向量库：PGVector（MVP）→ Milvus（扩展）

---

# 4️⃣ 数据闭环系统（核心竞争力）

## 🔄 数据流闭环

```
用户输入（饮食/体重/运动）
        ↓
系统分析（Agent）
        ↓
生成策略（饮食+运动）
        ↓
用户执行
        ↓
行为数据反馈
        ↓
Reflect Agent优化
```

---

## 🎯 核心能力

* 动态热量调整（TDEE自适应）
* 平台期检测（趋势分析）
* 个性化策略演化

---

# 5️⃣ 功能模块设计

---

## 5.1 用户画像模块

### 输入

* 身高 / 体重 / 年龄 / 性别
* 目标体重
* 生活习惯（作息/运动频率）

### 输出

* 用户Profile（JSON）
* 减脂阶段（初期/平台期/稳定期）

---

## 5.2 身体指标计算模块

### 核心指标

* BMI（体型评估）
* BMR（基础代谢）
* TDEE（总消耗）

👉 用于所有Agent决策基础

---

## 5.3 饮食决策模块（Diet Agent）

### 功能

* 每日热量目标
* 三餐推荐
* 外卖替代策略

### 特点

* 结合RAG动态生成
* 支持“现实约束”（如没时间做饭）

---

## 5.4 运动决策模块（Workout Agent）

### 功能

* 周训练计划
* 每日动作推荐
* 强度动态调整

---

## 5.5 行为干预模块（⭐核心亮点）

### 触发机制

| 场景   | 条件  |
| ---- | --- |
| 不运动  | ≥3天 |
| 热量超标 | 当日  |
| 情绪进食 | 高频  |

---

### 输出

* AI劝导（人格化）
* 替代行为建议（如低卡零食/轻运动）

👉 面试加分点：**“AI不只是建议，而是改变行为”**

---

## 5.6 进度分析模块（Progress Agent）

### 功能

* 体重趋势分析
* 平台期检测
* 目标偏差评估

---

## 5.7 策略优化模块（Reflect Agent）

### 功能

* 判断当前策略是否有效
* 自动调整：

  * 热量目标
  * 训练强度
  * 干预频率

---

## 5.8 决策中枢（Planner Agent）

### 职责

* Agent调度
* 决策整合
* 输出最终计划

---
